League of Legends Pick-Ban Optimizer

Attempting to solve the pick-ban phase using statistics

Posted on January 15, 2016

The pick-ban phase in League of Legends has always been an interesting mini-game. Given a 100+ champion pool, the vast combination of team composition provides a dynamic and challenging phase of the game. As the LoL moves to a 10-champion ban mode in the future, the complexity of the problem also increases. The purpose of this exercise is to attempt to facilitate the decision making process behind the pick-ban phase using statistics.

Demo

Demo and additional details can be found here.

When the opponent team first picks Ryze, we should consider playing a team comp with Vi or Wukong.

Data

In the current build, historical data consists of 10,000 games obtained from Riot API as of patch 5.13. I'm in the process of updating this process so that the model automatically recalibrates with the latest data every month. Ideally, the model should be able to filter by different game modes, player ranks, game time, and other attributes.

In building a model, data takes up 80% of the time, but gets 20% of the fame.

Methodology

The general methodology used in rank ordering champions is as follows.

  1. Filter the data set so that it includes all games (identified by gameid) with champions that have been picked by the two teams.
  2. Filter the data set so that it includes all opponent champions on one team, and all friendly champions on the opposite team.
  3. Calculate win rate per champion in the filtered data set.
